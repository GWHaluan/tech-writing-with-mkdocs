Schema Definitions

eventBody: object

The request parameters for generating a transactional questionnaire link

type: string

The event type associated with this link.

customer: object

The customer to be invited to leave a review.

firstName: string

The customer's first name.

lastName: string

The customer's last name.

email: string

The customer's email address.

mobile: string

The customer's mobile phone number.

address: string

The customer's address.

preferredCommunicationChannel: string EMAIL, SMS EMAIL

What communication channel is preferred by a customer

channel: object

The channel which will be associated with the review.

id: string

The unique identifier of the channel.

type: string user_defined, etrusted etrusted

The type of the id used to refer to the channel.

transaction: object

The transaction to be reviewed.

reference: string

An external reference to the transaction.

date: string (date-time)

The date when the transaction occurred.

system: string

The system name is the technical system name, in which this link will be imbedded. If problem occur, eTrusted will communicate this system name to the integrator for further analysis. Therefore the integrator should use precise names to differentiate his technical systems.

systemVersion: string

The system version is the version of the integrator system. This offers organizations the possibility to use different versions of the same integration without putting this information into the system name.

metadata: object

The metadata associated with the review.
